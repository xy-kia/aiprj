**核心架构代码示例**：
```python
# FastAPI主应用
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from app.api import parse, search, questions, evaluate

app = FastAPI(title="学生求职AI助手", version="1.0.0")

# CORS配置
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],  # 前端开发地址
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 注册路由
app.include_router(parse.router, prefix="/api/v1")
app.include_router(search.router, prefix="/api/v1")
app.include_router(questions.router, prefix="/api/v1")
app.include_router(evaluate.router, prefix="/api/v1")
```

**API接口清单**：
- `POST /api/v1/parse-intent` - 意向解析接口
- `POST /api/v1/search-jobs` - 岗位搜索接口
- `POST /api/v1/generate-questions` - 问题生成接口
- `POST /api/v1/evaluate-answer` - 回答评估接口
- `GET /health` - 健康检查接口
- `GET /docs` - OpenAPI文档（自动生成）

## 开发规范

### 前后端协作规范
1. **接口契约先行**：使用OpenAPI 3.0定义接口，前后端基于契约并行开发
2. **Mock数据支持**：前端开发阶段使用Mock.js模拟API响应
3. **联调时间点**：每周三、周五下午固定联调时间
4. **错误码统一**：定义全局错误码规范，前后端保持一致

### 代码质量要求
1. **TypeScript类型安全**：前端全面使用TypeScript，避免any类型
2. **单元测试覆盖率**：后端≥80%，前端≥70%
3. **E2E测试**：核心用户流程必须有端到端测试
4. **代码审查**：所有代码必须经过至少1人审查

### 部署要求
1. **容器化**：所有服务必须支持Docker容器化部署
2. **健康检查**：每个服务必须提供健康检查接口
3. **日志标准化**：使用JSON格式日志，包含请求ID追踪
4. **监控埋点**：关键业务指标必须埋点监控
