## 核心功能详述

### 2.1 节点1：意向解析与关键词生成

#### 功能描述
接收学生自然语言输入的岗位意向，解析并生成结构化搜索关键词。

#### 输入
- 学生自然语言描述（文本）

#### 输出
- 结构化关键词JSON

#### 功能流程
```
1. 接收用户输入
2. 文本预处理（清洗、分词）
3. 实体识别（岗位、技能、地点、学历、经验）
4. 关键词扩展（同义词、关联词）
5. 缺失值填充（默认学历/城市）
6. 输出结构化关键词
```

#### 技术实现
| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 分词 | jieba | 中文分词，支持自定义词典 |
| 实体识别 | 规则+词典匹配 | 基于技能库、岗位库匹配 |
| 关键词扩展 | 同义词库 | 技能映射库查询 |
| 缺失填充 | 默认值策略 | 学历默认本科，城市默认一线 |

#### 接口定义
```
POST /api/v1/parse-intent
Request:
{
  "raw_input": "我想找一份2025年暑假的产品经理实习..."
}

Response:
{

  "data": {
    "keywords": {
      "job_type": ["产品经理", "产品助理"],
      "skills": ["Axure", "数据分析"],
      "cities": ["上海", "杭州"],
      "education": "本科",
      "experience": "实习"
    },
    "parsed_entities": [...],
    "confidence": 0.92
  }
}
```

### 2.2 节点2：岗位搜索与匹配

#### 功能描述
基于关键词爬取四大招聘平台数据，计算匹配度，返回排序后的岗位列表。

#### 输入
- 结构化关键词
- 学生简历（可选）

#### 输出
- 岗位列表（含匹配度评分）

#### 功能流程
```
1. 并行触发多平台爬虫
2. 搜索结果页爬取（岗位列表）
3. 详情页爬取（完整JD）
4. 数据清洗与标准化
5. 匹配度计算（简历 vs JD）
6. 去重处理
7. 排序与截断
8. 返回结果
```

#### 爬虫配置
| 平台 | 框架 | 反爬策略 | 请求间隔 |
|------|------|----------|----------|
| BOSS直聘 | Playwright | User-Agent轮换、代理IP | 1-3秒 |
| 猎聘 | Playwright | 随机延时、Cookie管理 | 2-5秒 |
| 智联招聘 | Requests+JSON | 请求头伪装 | 1-2秒 |
| 前程无忧 | Requests+lxml | Referer设置 | 1-2秒 |

#### 匹配度算法
```python
# 伪代码
def calculate_match_score(resume, jd):
    # 1. 文本向量化
    resume_vec = vectorize(resume)
    jd_vec = vectorize(jd)
    
    # 2. 计算余弦相似度
    base_score = cosine_similarity(resume_vec, jd_vec)
    
    # 3. 关键词匹配加分
    keyword_bonus = keyword_match(resume, jd)
    
    # 4. 加权计算
    final_score = base_score * 0.6 + keyword_bonus * 0.4
    
    return final_score
```

### 2.3 节点3：个性化问题生成

#### 功能描述
基于选定岗位的JD和学生简历，生成定制化的面试问题（一般版+高阶版）。

#### 输入
- 选定岗位信息（含JD全文）
- 学生简历摘要

#### 输出
- 两套问题集合（一般版3-5个 + 高阶版3-5个）

#### 功能流程
```
1. 解析JD，提取关键能力要求
2. 分析简历，提取经历与技能
3. 召回同类岗位问题模板
4. 构造LLM Prompt
5. 调用LLM生成问题
6. 质量问题过滤
7. 格式化输出
```

#### Prompt模板
```

【岗位信息】
岗位名称：{job_title}
JD内容：{jd_text}

【候选人背景】
技能：{skills}
经历：{experiences}

【要求】
1. 设计3-5个一般实习版问题：侧重基础技能、执行力、学习能力

3. 每个问题需标注考察的能力点
4. 问题需与JD要求紧密相关

【输出格式】

  "general": [...],
  "advanced": [...]
```

### 2.4 节点4：模拟回答评估

#### 功能描述
接收学生对问题的模拟回答，进行多维度评估并输出评分与建议。

#### 输入
- 问题信息（含JD参考）
- 学生回答文本

#### 输出
- 评分结果
- 改进建议
- 决策建议

#### 功能流程
```
1. 接收学生回答
2. 内容质量检查（长度、相关性）
3. 关键词匹配分析
4. 语义相似度分析
5. 多维度评分
6. 生成改进建议
7. 综合决策
8. 输出评估报告
```

#### 评分算法
```python
# 伪代码
def evaluate_answer(answer, question, jd_requirements):
    scores = {}
    
    # 1. 岗位技能覆盖度
    matched_keywords = [kw for kw in jd_requirements if kw in answer]
    scores['skill_coverage'] = len(matched_keywords) / len(jd_requirements) * 10
    
    # 2. 案例具体性（基于关键词检测）
    specificity_markers = ['项目', '负责', '完成', '实现']
    scores['specificity'] = count_markers(answer, specificity_markers)
    
    # 3. 逻辑表达（基于结构检测）
    structure_score = detect_star_structure(answer)
    scores['logic'] = structure_score
    
    # 4. 量化成果（数字检测）
    has_numbers = bool(re.search(r'\d+%?|\d+个|\d+万', answer))
    scores['quantification'] = 8 if has_numbers else 4
    
    # 综合分
    overall = weighted_average(scores)
    
    # 决策
    decision = '通过' if overall >= 7 else ('待提升' if overall >= 5 else '不通过')
    
    return {
        'scores': scores,
        'overall': overall,
        'decision': decision
    }
```

## 数据库设计需求规划

### 3.1 数据库存储需求

| 数据类型 | 存储方案 | 说明 |
|----------|----------|------|
| 会话数据 | Redis | 临时存储，TTL 24小时 |
| 岗位数据 | MySQL + 缓存 | 结构化存储，7天过期 |
| 评估记录 | MySQL | 业务数据，90天保留 |
| 知识库 | JSON文件/ES | 静态数据，版本管理 |
| 向量数据 | FAISS/Milvus | 相似度检索 |

### 3.2 核心表结构

#### 会话表（session）
```sql
CREATE TABLE session (
    session_id VARCHAR(64) PRIMARY KEY,
    student_id VARCHAR(64),
    resume_text TEXT,
    target_keywords JSON,
    status ENUM('active', 'expired'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    expire_at TIMESTAMP
);
```

#### 岗位表（jobs）
```sql
CREATE TABLE jobs (
    job_id VARCHAR(64) PRIMARY KEY,
    platform VARCHAR(32),
    company_name VARCHAR(128),
    job_title VARCHAR(128),
    location VARCHAR(64),
    salary_min INT,
    salary_max INT,
    salary_unit VARCHAR(16),
    jd_text TEXT,
    jd_url VARCHAR(512),
    publish_time DATE,
    crawl_time TIMESTAMP,
    INDEX idx_company (company_name),
    INDEX idx_location (location),
    INDEX idx_publish_time (publish_time)
);
```

#### 评估记录表（evaluations）
```sql
CREATE TABLE evaluations (
    evaluation_id VARCHAR(64) PRIMARY KEY,
    session_id VARCHAR(64),
    job_id VARCHAR(64),
    question_id VARCHAR(64),
    answer_text TEXT,
    scores JSON,
    overall_score DECIMAL(3,1),
    decision VARCHAR(16),
    created_at TIMESTAMP,
    INDEX idx_session (session_id),
    INDEX idx_job (job_id)
);
```
